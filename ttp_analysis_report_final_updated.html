Hypothesis & analysis approach

The question we are trying to answer and why it is hard to prove.

The hypothesis
Publishing a funnel during the trial period independently causes higher trial-to-paid conversion — not because publishers are more engaged users overall, but because the act of getting a funnel live creates real value that drives the decision to pay.
The challenge — why this is hard to prove
The engagement confounder
What the raw data shows
Publishers convert at 72% vs non-publishers at 48% — a massive +24pp gap. But this is misleading because publishers are also running more workflows, sending more emails, creating more contacts. They're just more engaged users across the board.
What we need to know
Is the higher conversion because of the funnel publish itself? Or would these companies have converted anyway because they're generally more committed users? Answering this requires isolating the funnel effect from everything else.

The approach

4 methods to isolate the funnel effect

If all four methods converge on the same answer despite using completely different statistical approaches, the finding is credible.

Why this analysis: The confounder check confirmed publishers are more active across every other product. So the raw +25pp gap includes both the funnel effect and the engagement effect. We need methods that strip away the engagement difference and isolate what funnel publishing alone contributes to conversion.

Bottom line: All four methods agree on direction — publishing a funnel during trial independently lifts conversion. The controlled estimates (PSM, logistic regression) converge at +2–4pp. The funnel-only cohort shows a larger gap (+23–35pp) because it compares against a lower-intent control group. Isolating the effect · Method 1

Funnel-only users

This removes the engagement confounder by design.

Why this analysis: If we restrict to companies that used GHL exclusively for funnels during trial — zero workflows, contacts, emails, SMS, opportunities, forms, calls, or calendar events — then the engagement confounder is removed by construction. Everyone has equally zero other-product activity. Any conversion difference between publishers and non-publishers must come from publishing itself.
Why this matters: Publishers convert higher partly because they're more engaged overall. By restricting to companies with zero other-product activity, we strip that explanation away. Whatever gap remains is attributable to funnel publishing itself — not general engagement.

-- Stratified Mantel-Haenszel weighted comparison
-- US only · non-snapshot funnels only
-- Strata: plan × niche × primary_use × client_size × trial_experience × engagement_bucket
WITH trials AS (
  SELECT
    CUSTOMER_METADATA_HL_COMPANY_ID AS company_id,
    MIN(CONVERTED_TRIAL_START_DATE::DATE) AS trial_start,
    MAX(CONVERTED_TRIAL_END_DATE::DATE) AS trial_end,
    MAX(CASE WHEN CM_CUSTOMER_STATUS != 'Lead' THEN 1 ELSE 0 END) AS converted,
    MAX(SI_PLAN_ID) AS plan_id,
    MAX(BUSINESS_CATEGORY) AS niche,
    MAX(CUSTOMER_METADATA_AGENCY_PRIMARY_USE_CONSOLIDATED) AS primary_use,
    MAX(CUSTOMER_METADATA_NUMBER_OF_CLIENTS) AS client_size,
    MAX(TRIAL_EXPERIENCE) AS trial_experience
  FROM PROD_HIGHLEVEL_DWH.MOZART.NEW_TRIALS
  WHERE CONVERTED_TRIAL_START_DATE >= '2026-04-01'
    AND CONVERTED_TRIAL_START_DATE < '2026-05-01'
    AND CONVERTED_TRIAL_END_DATE IS NOT NULL
    AND CUSTOMER_METADATA_HL_COMPANY_ID IS NOT NULL
    AND COUNTRY_NAME = 'United States of America'
  GROUP BY 1
),
-- ... (CTEs for company_locations, loc_funnel, engagement_score as in overview) ...
-- engagement_score = count of products activated during trial (workflow, email, contact, etc.)
features AS (
  SELECT t.*, lf.bucket,
    CASE WHEN engagement_score <= 1 THEN 'low'
         WHEN engagement_score <= 4 THEN 'medium'
         ELSE 'high' END AS engagement_bucket
  FROM trials_with_engagement t
  JOIN loc_funnel lf ON t.company_id = lf.COMPANY_ID
),
stratified AS (
  SELECT *,
    CONCAT_WS('|',
      plan_id, COALESCE(niche,'Unknown'), primary_use,
      client_size, trial_experience,
      engagement_bucket
    ) AS stratum
  FROM features
),
stratum_rates AS (
  SELECT stratum,
    SUM(CASE WHEN bucket='A' THEN 1 ELSE 0 END) AS n_A,
    SUM(CASE WHEN bucket='B' THEN 1 ELSE 0 END) AS n_B,
    AVG(CASE WHEN bucket='A' THEN converted END) AS conv_A,
    AVG(CASE WHEN bucket='B' THEN converted END) AS conv_B
  FROM stratified
  GROUP BY 1
  HAVING n_A >= 3 AND n_B >= 3
)
-- Mantel-Haenszel weighted average
SELECT
  SUM((conv_A - conv_B) * (n_A * n_B) / NULLIF(n_A + n_B, 0)) /
  NULLIF(SUM((n_A * n_B) / NULLIF(n_A + n_B, 0)), 0) AS weighted_att_A_vs_B
FROM stratum_rates

def run_psm_cohort(df, treat_bkt, control_bkt, label, caliper=0.05):
    """
    Train logistic regression on THIS cohort only.
    Match 1:1 by propensity score within caliper.
    Return ATT and diagnostics.
    """
    df_sub = df[df['bkt'].isin([treat_bkt, control_bkt])].copy().reset_index(drop=True)
    treat  = (df_sub['bkt'] == treat_bkt).astype(int).values

    # One-hot encode categoricals — fit/transform on this cohort only
    df_enc = pd.get_dummies(df_sub, columns=['niche', 'puse'], drop_first=False)
    feature_cols = [c for c in df_enc.columns if c not in ['y', 'bkt']]
    X = df_enc[feature_cols].astype(float).fillna(0)

    # Scale on this cohort only
    scaler   = StandardScaler()
    X_scaled = scaler.fit_transform(X)

    # Logistic regression trained on this cohort only
    lr = LogisticRegression(max_iter=1000, C=1.0, random_state=42)
    lr.fit(X_scaled, treat)
    ps = lr.predict_proba(X_scaled)[:, 1]
    df_sub['ps'] = ps

    # Split by treatment / control
    treat_idx   = df_sub[df_sub['bkt'] == treat_bkt].index.tolist()
    control_idx = df_sub[df_sub['bkt'] == control_bkt].index.tolist()

    # 1:1 nearest-neighbour matching
    ps_treat   = df_sub.loc[treat_idx,   'ps'].values.reshape(-1, 1)
    ps_control = df_sub.loc[control_idx, 'ps'].values.reshape(-1, 1)

    nn = NearestNeighbors(n_neighbors=1, metric='euclidean')
    nn.fit(ps_control)
    distances, indices = nn.kneighbors(ps_treat)

    matched_treat, matched_control = [], []
    for i, (d, idx) in enumerate(zip(distances.flatten(), indices.flatten())):
        if d <= caliper:
            matched_treat.append(treat_idx[i])
            matched_control.append(control_idx[idx])

    # Auto-relax caliper if too few matches
    if len(matched_treat) < 50:
        caliper = 0.10
        matched_treat, matched_control = [], []
        for i, (d, idx) in enumerate(zip(distances.flatten(), indices.flatten())):
            if d <= caliper:
                matched_treat.append(treat_idx[i])
                matched_control.append(control_idx[idx])

    if not matched_treat:
        return None

    # Core metrics
    conv_treat   = df_sub.loc[matched_treat,   'y'].mean()
    conv_control = df_sub.loc[matched_control, 'y'].mean()
    att          = conv_treat - conv_control
    raw_treat    = df_sub[df_sub['bkt'] == treat_bkt]['y'].mean()
    raw_control  = df_sub[df_sub['bkt'] == control_bkt]['y'].mean()
    raw_gap      = raw_treat - raw_control
    conf_pct     = (raw_gap - att) / raw_gap * 100 if raw_gap != 0 else 0
    match_rate   = len(matched_treat) / len(treat_idx) * 100

    # SMD after matching (key balance check)
    numeric_cols = ['plan','client_size','trial_exp','trial_len','snap1',
                    'we','es','cc','ce','ss','da','pc','dc']
    smd_rows = []
    for col in numeric_cols:
        if col not in df_sub.columns:
            continue
        t_vals = df_sub.loc[matched_treat,   col]
        c_vals = df_sub.loc[matched_control, col]
        pooled = np.sqrt((t_vals.var() + c_vals.var()) / 2)
        smd    = abs(t_vals.mean() - c_vals.mean()) / pooled if pooled > 0 else 0
        smd_rows.append({'var': col, 'smd': smd})
    smd_df      = pd.DataFrame(smd_rows)
    mean_smd    = smd_df['smd'].mean()
    max_smd     = smd_df['smd'].max()
    max_smd_var = smd_df.loc[smd_df['smd'].idxmax(), 'var']

    return {
        'label':             label,
        'treat_bkt':         treat_bkt,
        'control_bkt':       control_bkt,
        'n_treat':           len(treat_idx),
        'n_control':         len(control_idx),
        'matched_pairs':     len(matched_treat),
        'match_rate':        round(match_rate, 1),
        'raw_treat_pct':     round(raw_treat * 100, 1),
        'raw_control_pct':   round(raw_control * 100, 1),
        'raw_gap_pp':        round(raw_gap * 100, 1),
        'conv_treat_pct':    round(conv_treat * 100, 1),
        'conv_control_pct':  round(conv_control * 100, 1),
        'att_pp':            round(att * 100, 1),
        'conf_pct':          round(conf_pct, 0),
        'caliper':           caliper,
        'mean_smd':          round(mean_smd, 3),
        'max_smd':           round(max_smd, 3),
        'max_smd_var':       max_smd_var,
        'ps_balance':        round(abs(df_sub.loc[matched_treat,'ps'].mean() -
                                       df_sub.loc[matched_control,'ps'].mean()), 4),
    }

WITH trials AS (
  SELECT
    CUSTOMER_METADATA_HL_COMPANY_ID AS company_id,
    MIN(CONVERTED_TRIAL_START_DATE::DATE) AS trial_start,
    MAX(CONVERTED_TRIAL_END_DATE::DATE) AS trial_end,
    MAX(CASE WHEN CM_CUSTOMER_STATUS != 'Lead' THEN 1 ELSE 0 END) AS converted,
    DATE_TRUNC('month', MIN(CONVERTED_TRIAL_START_DATE::DATE))::DATE AS cohort_month
  FROM PROD_HIGHLEVEL_DWH.MOZART.NEW_TRIALS
  WHERE CONVERTED_TRIAL_START_DATE >= '2026-02-01'
    AND CONVERTED_TRIAL_START_DATE < '2026-05-01'
    AND CONVERTED_TRIAL_END_DATE IS NOT NULL
    AND CUSTOMER_METADATA_HL_COMPANY_ID IS NOT NULL
    AND COUNTRY_NAME = 'United States of America'
  GROUP BY 1
),
cl AS (
  SELECT DISTINCT COMPANY_ID, LOCATION_ID
  FROM HL_PRODUCT_DB.PRODUCT_ANALYTICS.ACCOUNT_DAILY_METRICS_REPORT
  WHERE COMPANY_ID IS NOT NULL AND LOCATION_ID IS NOT NULL
),
funnels_published AS (
  SELECT cl.COMPANY_ID,
    COUNT(DISTINCT fm.FUNNEL_ID) AS num_published
  FROM cl
  JOIN trials t ON cl.COMPANY_ID = t.company_id
  JOIN HL_PRODUCT_DB.PRODUCT_LEADGEN.FUNNEL_METAS fm ON fm.LOCATION_ID = cl.LOCATION_ID
  WHERE LOWER(fm.FUNNEL_TYPE) IN ('funnel','website')
    AND fm.IS_DELETED = FALSE
    AND fm.SOURCE_TYPE != 'snapshot'
    AND fm.DOMAIN_ATTACHED_AT::DATE BETWEEN t.trial_start AND t.trial_end
  GROUP BY cl.COMPANY_ID
)
SELECT
  t.cohort_month,
  CASE
    WHEN COALESCE(fp.num_published, 0) = 0 THEN 'C) No funnel published'
    WHEN fp.num_published = 1              THEN 'A1) Published 1 funnel'
    WHEN fp.num_published = 2              THEN 'A2) Published 2 funnels'
    ELSE                                        'A3) Published 3+ funnels'
  END AS bucket,
  COUNT(*) AS companies,
  ROUND(AVG(t.converted) * 100, 2) AS conversion_rate_pct
FROM trials t
LEFT JOIN funnels_published fp ON t.company_id = fp.COMPANY_ID
GROUP BY 1, 2
ORDER BY 1, 2

How to interpret this
Why same-day is lowest
Same-day publishers published within hours of starting their trial — before deeply exploring the platform. This speed may reflect lower deliberate intent: they set something up quickly but may not have invested enough in understanding its value, which could explain the lower conversion rate.
Why 8–14 days is the sweet spot
Companies that publish 8–14 days in have explored the platform, understood its value, and made a deliberate decision to build and publish a funnel. The time investment signals genuine intent. They also still have 6–22 days of trial left to experience the funnel working — long enough to see leads come in and decide to pay. Late publishers (15+) may not have that runway.
Implication for onboarding design: Consider a nudge at days 3–5 ("your trial is going well — ready to publish your first funnel?") that guides users toward the 8–14 day publish window where conversion is highest.

-- Time to publish: conversion by days-to-first-publish bucket
-- Feb, Mar, Apr 2026 · US only · non-snapshot funnels only
WITH trials AS (
  SELECT
    CUSTOMER_METADATA_HL_COMPANY_ID AS company_id,
    MIN(CONVERTED_TRIAL_START_DATE::DATE) AS trial_start,
    MAX(CONVERTED_TRIAL_END_DATE::DATE) AS trial_end,
    MAX(CASE WHEN CM_CUSTOMER_STATUS != 'Lead' THEN 1 ELSE 0 END) AS converted,
    DATE_TRUNC('month', MIN(CONVERTED_TRIAL_START_DATE::DATE))::DATE AS cohort_month
  FROM PROD_HIGHLEVEL_DWH.MOZART.NEW_TRIALS
  WHERE CONVERTED_TRIAL_START_DATE >= '2026-02-01'
    AND CONVERTED_TRIAL_START_DATE < '2026-05-01'
    AND CONVERTED_TRIAL_END_DATE IS NOT NULL
    AND CUSTOMER_METADATA_HL_COMPANY_ID IS NOT NULL
    AND COUNTRY_NAME = 'United States of America'
  GROUP BY 1
),
cl AS (
  SELECT DISTINCT COMPANY_ID, LOCATION_ID
  FROM HL_PRODUCT_DB.PRODUCT_ANALYTICS.ACCOUNT_DAILY_METRICS_REPORT
  WHERE COMPANY_ID IS NOT NULL AND LOCATION_ID IS NOT NULL
),
first_publish AS (
  SELECT
    cl.COMPANY_ID, t.trial_start, t.trial_end, t.converted, t.cohort_month,
    MIN(fm.DOMAIN_ATTACHED_AT::DATE) AS first_publish_date,
    DATEDIFF('day', t.trial_start, MIN(fm.DOMAIN_ATTACHED_AT::DATE)) AS days_to_publish
  FROM cl
  JOIN trials t ON cl.COMPANY_ID = t.company_id
  JOIN HL_PRODUCT_DB.PRODUCT_LEADGEN.FUNNEL_METAS fm ON fm.LOCATION_ID = cl.LOCATION_ID
  WHERE LOWER(fm.FUNNEL_TYPE) IN ('funnel','website')
    AND fm.IS_DELETED = FALSE
    AND fm.SOURCE_TYPE != 'snapshot'
    AND fm.DOMAIN_ATTACHED_AT::DATE BETWEEN t.trial_start AND t.trial_end
  GROUP BY cl.COMPANY_ID, t.trial_start, t.trial_end, t.converted, t.cohort_month
)
SELECT
  cohort_month,
  CASE
    WHEN days_to_publish = 0  THEN '0 — Same day'
    WHEN days_to_publish <= 3 THEN '1–3 days'
    WHEN days_to_publish <= 7 THEN '4–7 days'
    WHEN days_to_publish <= 14 THEN '8–14 days'
    ELSE '15+ days'
  END AS days_to_publish_bucket,
  COUNT(*) AS companies,
  ROUND(AVG(converted) * 100, 2) AS conversion_rate_pct
FROM first_publish
GROUP BY 1, 2
ORDER BY 1, MIN(days_to_publish)

-- Funnel publish propensity and conversion by niche
-- April 2026 · US only · non-snapshot funnels only
WITH trials AS (
  SELECT
    CUSTOMER_METADATA_HL_COMPANY_ID AS company_id,
    MIN(CONVERTED_TRIAL_START_DATE::DATE) AS trial_start,
    MAX(CONVERTED_TRIAL_END_DATE::DATE) AS trial_end,
    MAX(CASE WHEN CM_CUSTOMER_STATUS != 'Lead' THEN 1 ELSE 0 END) AS converted,
    MAX(BUSINESS_CATEGORY) AS niche
  FROM PROD_HIGHLEVEL_DWH.MOZART.NEW_TRIALS
  WHERE CONVERTED_TRIAL_START_DATE >= '2026-04-01'
    AND CONVERTED_TRIAL_START_DATE < '2026-05-01'
    AND CONVERTED_TRIAL_END_DATE IS NOT NULL
    AND CUSTOMER_METADATA_HL_COMPANY_ID IS NOT NULL
    AND COUNTRY_NAME = 'United States of America'
  GROUP BY 1
),
cl AS (
  SELECT DISTINCT COMPANY_ID, LOCATION_ID
  FROM HL_PRODUCT_DB.PRODUCT_ANALYTICS.ACCOUNT_DAILY_METRICS_REPORT
  WHERE COMPANY_ID IS NOT NULL AND LOCATION_ID IS NOT NULL
),
loc_funnel AS (
  SELECT cl.COMPANY_ID,
    MAX(CASE WHEN fm.CREATED_AT::DATE BETWEEN t.trial_start AND t.trial_end THEN 1 ELSE 0 END) AS cf,
    MAX(CASE WHEN fm.DOMAIN_ATTACHED_AT::DATE BETWEEN t.trial_start AND t.trial_end THEN 1 ELSE 0 END) AS pf
  FROM cl
  JOIN trials t ON cl.COMPANY_ID = t.company_id
  JOIN HL_PRODUCT_DB.PRODUCT_LEADGEN.FUNNEL_METAS fm ON fm.LOCATION_ID = cl.LOCATION_ID
  WHERE LOWER(fm.FUNNEL_TYPE) IN ('funnel','website')
    AND fm.IS_DELETED = FALSE
    AND fm.SOURCE_TYPE != 'snapshot'
  GROUP BY cl.COMPANY_ID
)
SELECT
  t.niche,
  COUNT(*) AS n,
  ROUND(AVG(CASE WHEN COALESCE(lf.pf,0)=1 THEN 1.0 ELSE 0 END)*100,1) AS publish_rate_pct,
  ROUND(AVG(CASE WHEN COALESCE(lf.cf,0)=1 AND COALESCE(lf.pf,0)=0 THEN 1.0 ELSE 0 END)*100,1) AS create_not_pub_pct,
  ROUND(AVG(CASE WHEN COALESCE(lf.cf,0)=0 THEN 1.0 ELSE 0 END)*100,1) AS no_funnel_pct,
  ROUND(AVG(CASE WHEN COALESCE(lf.pf,0)=1 THEN t.converted END)*100,1) AS conv_if_published
FROM trials t
LEFT JOIN loc_funnel lf ON t.company_id = lf.COMPANY_ID
GROUP BY 1
ORDER BY publish_rate_pct DESC
